## **Bonus Task: Interactive Meeting Playback UI**

### **Objective**

Create a simple interactive web interface that displays the meeting video with a synchronized, clickable transcript sidebar.

### **Requirements**

1. **Interface Layout**  
   * Video player on the main panel  
   * Transcript displayed in a scrollable sidebar  
   * Responsive design that works on desktop browsers  
2. **Core Functionality**  
   * Clicking on any transcript segment jumps the video to the corresponding timestamp  
   * Current transcript segment is highlighted as the video plays  
   * Auto-scrolling transcript that follows along with video playback  
3. **Technical Implementation**  
   * Use HTML, CSS, and JavaScript (frameworks like React are optional but welcomed)  
   * Implement a basic video player using HTML5 video element or a library like Video.js  
   * Parse and display the transcript with proper formatting and timestamps  
4. **Deliverables**  
   * HTML/CSS/JS code for the interactive interface  
   * Brief documentation explaining how to use the interface  
   * Include this in the same GitHub repository as the main challenge  
   * Optional: add a section to your video demo showing this interface in action

### **Notes**

* The interface doesn't need to be visually complex \- focus on functionality  
* You don't need to implement user authentication or cloud storage  
* The interface should load local files (video and transcript) that have been processed by your main solution

### **Example (for clarity)**

Similar to Zoom's meeting playback interface, where:

* The video plays in the main area  
* Transcript entries are displayed on the right  
* Clicking on transcript text jumps to that point in the recording  
* Current segment is highlighted as the video plays

This bonus task will demonstrate your frontend skills and ability to create practical user interfaces for data consumption.
